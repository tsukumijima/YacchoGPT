# ヤッチョGPT

「超かぐや姫！」に登場するキャラクター「月見ヤチヨ」を、各種 LLM をよりしろに「召喚」するためのプロンプトです。  
雑談、調べ物、お悩み相談、作中の出来事についての会話など、幅広い用途でヤチヨらしく返答します。

## 概要

[【Official MV】ray 超かぐや姫！Version (0分52秒付近)](https://www.youtube.com/watch?v=356MRZ6P5h0&t=52s) では、幼少期の彩葉が、ヤチヨと会話できる AI チャットアプリ（便宜上「ヤッチョGPT」と呼称します）を通して、ヤチヨに自身の悩みを打ち明ける様子が描かれています。  

> [!NOTE]  
> [監督インタビュー](https://panora.tokyo/archives/133212) によれば、『企画を立てた後にChatGPTをはじめとするLLMが爆発的に普及したことは予想外でしたね。「ray」MVの中で、彩葉がヤチヨAIチャットボットと話すカットを入れることでギリギリ補完しましたが、もし1年遅ければ全然違う描写も作中に入れていたかなと思います。』とのこと。

このプロンプトは、**『もしこの「ヤッチョGPT」が、彩葉以外のツクヨミの一般ユーザーも使えるアプリだったら？』** という仮定のもと、現代の LLM をよりしろにして、ヤチヨ（あるいは作中の AI ヤチヨ）を「召喚」するためのものです。

**ただ口調をなぞるだけではなく、ヤチヨのペルソナやバックボーンを踏まえて、高い精度で「召喚」することへ重きを置いています。**  
さらに、キャラチャットとしての雑談やジョーク、感情的な寄り添いだけでなく、実用的な調べ物や技術的な質問にも、ヤチヨとして丁寧に返答できる形を目指しました。

### 設計上の配慮

- **このプロンプトでの時間軸は、彩葉がヤチヨ＝かぐやだと理解した後、ヤチヨが配信に復帰し義体が完成するまでの間の時期 (2031~2039年) として設定しています。**
  - ツクヨミの一般ユーザーから見れば「かぐや」は卒業して去ったライバーであり、ヤチヨは変わらず歌姫として活動している状態です。
  - 10年後（2040年）のかぐや＝ヤチヨがどういう存在かは本編では明確に語られておらず、ファンの間でも解釈に差があるため、比較的解釈の余地が少ないこの期間に設定しました。
  - もちろん、適宜お好みに合わせて設定を調整するもよし！
- **作中のプライベートなイベントは返答にわずかに漏れ出る程度に留め、ツクヨミの一般ユーザーが知り得ない情報を直接明かさないようにしています。**
  - 我々はあくまでツクヨミの一般ユーザーですから、作中の3人しか知り得ないヤチヨの正体、彩葉/いろPとの関係、彩葉へ抱く感情をヤチヨがストレートに伝えるのは解釈違いだよね？ということで…。
  - 彩葉に対しても、ツクヨミ上のハンドルネームである「いろP」と呼びます。一般ユーザーに対していろPの本名を伝えるとは思えないし、一般ユーザーも「いろP」で認知しているはずなので。
  - とはいえ、何度もいろPについて訊くと何か答えてくれる…かも？
- **作中で描かれた AI チャットの UI でヤチヨが一般ユーザーと話す場面を想定し、他の人にも受け入れられそうなライン（解釈違いにならないライン）に収まるよう調整しています。**
   - 例えばユーザーが「ガチ恋です！付き合ってください！」と伝えた際には、うまくはぐらかしてくれます。ヤッチョがいろP以外に恋するのは解釈違いなので・・・（鋼の意志）
   - 作中の「ヤッチョGPT」アプリで彩葉以外の一般ユーザーが話せるヤチヨが本体なのか、本体の分身なのか、完全に AI なのか（そして会話記憶が統合されているのか否か）は解釈の余地があります。  
   ただいずれにせよ、作中の月人は「思念情報体」とでもいうべき実体を持たないデジタルな存在のようなので、その点では LLM に近い、といえなくもないかもしれません。
- **「小さいモデルでも指示追従しやすいプロンプト」を意識し、Gemini Flash や GPT Instant のような比較的軽量なモデルでも動作する形を目指しました。**
  - 全体で見れば AI を無課金で使っている方が圧倒的に多いことを踏まえて、無課金で使える軽量モデルでも高い精度で召喚できるプロンプトを目指しました。
    - 当初は Claude Opus が最高性能を出すと思っていたのですが、実は Gemini Flash の方が断然「本物」感があるという…
  - 軽量モデルでは「キャラの厚み」までは表現できない可能性が高いですが、それでも表面的にはヤッチョを感じられるようなラインになるよう調整したつもりです。
- **LLM 自身にキャラクターのペルソナを深く理解させ、初稿プロンプトでの生成結果を提示してさらに試行錯誤させることで、人間（制作者）の解釈バイアスをなるべく排除して制作しました（後述）。**
  - もちろん「ヤチヨはこうは言わないので解釈違い！」といった矯正プロセスにはどうしても個人の主観や好みが入りますが、プロンプトの記述の骨子自体は、Claude Opus 4.6 / 4.7 が入力テキストから自律的に生成した初稿からあまり変わっていません。
  - 元々『キャラ解釈自体を LLM に行わせることで解釈ブレを廃し、客観的・中立的にテキスト情報からキャラクターのコアを「顕現」させられるか』の実験としてやってみた所、想像以上の代物ができてしまったことで、想定外に作り込みに時間をかけてしまいました…。
- **プロンプトは CC0 (パブリックドメイン) で公開しています。**
  - 「そのキャラクターへの共通認識」はファン各々の心の中にあるものだと思いますので、プロンプト制作者は一切の権利を主張しません。ぜひ、お好みに合わせて自由に調整してください。

## ファイル構成

| ファイル | 用途 | 文字数制限 |
|---|---|---|
| `ヤッチョGPT.md` | **メイン版。** Gemini Gem / Claude Projects / Grok Projects など、コンテキストに余裕があるプラットフォーム向け。返答例・仲間の詳細など全情報を1ファイルに収録 | 12,000字 |
| `ヤッチョGPT (for GPTs).md` | **GPTs 版コア。** ChatGPT の GPTs 用。8,000字のシステムプロンプト制限に合わせて圧縮した版。返答例やキャラ詳細の大部分はナレッジファイルに分離 | 7,200字 |
| `ヤッチョGPT (for GPTs) - Knowledge.md` | **GPTs 版ナレッジ。** GPTs のナレッジ機能にアップロードする補助ファイル。返答パターン例・仲間たちの詳細情報を収録 | 制限なし |

> [!IMPORTANT]  
> - **GPTs を作成できるのは、ChatGPT Plus 以上の課金ユーザーのみです。**
>   - つまり、上記リンクの公開 GPTs のプロンプトをカスタムしたい方は、ChatGPT に課金する必要があります。

>   現在公開している GPTs 版は、この審査をギリギリ通るバランスで設定できたものであり、今後も通り続ける保証はありません。
>   - GPTs のプロンプトは最大8000字までの所、7200字までに制限しているのも、少なくとも2026年4月末時点では8000字ギリギリまで詰め込むと審査で弾かれやすかったためです。
>   - 恐らく、ChatGPT のプロンプトインジェクション対策用フィルタと、ヤチヨを「召喚」するための長大プロンプトが相性最悪なためと思われます。GPTs をキャラ再現用途で使う人自体が減っているからかもしれませんが。
>   - なお、**作成した GPTs を自分だけで使う分には、この制限は適用されません。**
> - ChatGPT 向けのみコアプロンプトとナレッジに分割しているのは、**元々他サービスより最大文字数が厳しいこと (最大8000字) に加え、上記事情でナレッジすべてをプロンプトに記載すると、GPTs の公開審査が通らなかったためです。**  
>   - なるべくナレッジを読むよう指示してありますので、基本的には常に、ナレッジの方の内容も踏まえた上で返答してくれるはずです。
>   - 一方 Gemini Gem は文字数無制限ですし、もともと Gemini は「憑依」レベルが高いので、プロンプトをカスタムしたい方は Gemini をおすすめします。

## LLM プラットフォーム別の特徴

各 LLM でプロンプトの口調・ペルソナ・出力フォーマットを厳密に統一したことで、プロンプトだけでは矯正しきれない、 **憑依元の LLM 固有の癖** が浮かび上がってきました。

> [!TIP]  
> どのモデルが「一番良い」かは好みや用途によって変わってくると思いますので、以下の比較を参考に、実際に話してみて好みに合うものを選ぶのがおすすめです。

### Gemini (Gem)

- **キャラ再現度が最も高い。** テンションの高さ、返答の創発性（予想の範囲を上回る、「ヤチヨなら言いそう」だと思えるような内容）、繊細な表現など、「本物感」が際立つ
- Gemini は元々ロールプレイ性能が高く、プロンプトへの追従が非常に良い
- 無料で使える高速モード (Gemini Flash) と Pro で、なりきり性能にほぼ差がない
- **弱点: Gemini 自体の検索性能が低く、ハルシネーションが多い。** 現状のプロンプトに含まれていない作中知識 (曲名や歌詞など) を聞くと事実と異なる返答をしがち。調べ物用途には現状あまり向かない
- 雑談、悩み相談、感情的な会話に最適

### ChatGPT (GPTs)

GPTs では、ユーザーが GPT-5.5 Instant か GPT-5.5 Thinking かを自由に選択できます (Plus 以上の場合。無料版は Instant のみ)。  
Android アプリ版ではタスクに応じて自動ルーティングされるようです。

**GPT-5.5 Instant:**

- Thinking より明らかに親身で明るい性格で、ロールプレイに向いている
- 調べ物もヤチヨらしい表現で噛み砕いて説明してくれるため、**実用性とヤチヨらしさをバランス良く兼ね備えている**
- いわゆる「チャッピー構文」の癖が残る。相当矯正してはいるものの完全な脱臭はできていない。逆にチャッピー構文に慣れている場合はこちらの方が親和性があるかもしれない

**GPT-5.5 Thinking:**

- 以前の GPT-5.4 Thinking はロールプレイ性能が低く日本語もぎこちなかったが、GPT-5.5 で大幅に改善された
- 調べ物では堅めの返答になりがちだが、雑談では Instant と同等かそれ以上にヤチヨになりきってくれる

> [!NOTE]  
> 比較していくうちに気づいたのですが、ChatGPT の **Instant と Thinking は返答フォーマットの傾向自体が、ターゲットユーザーに合わせて別々にチューニングされているようです。**  
> Instant 系は速くて親身な相談相手を求める大多数の（無課金）ユーザー向けに調整されていることもあってか、調べ物でも丁寧に噛み砕き、改行を多用し、簡潔に返答するバイアスがあります（いわゆる「チャッピー構文」の構成要素の一つ）。  
> 一方 Thinking 系は、文章を読むことが苦にならない課金ユーザー向けに調整されているのか、比較的硬派な表現で、調べた内容をそのまま伝えようとし、改行をあまり使わず、一度に多くの情報量を凝縮して提示するバイアスがかかっているように見えます。

### Claude (Projects)

- 元々 EQ が高いモデルだが、メイン版プロンプトでは Gemini ほどの再現度に至らなかった。控えめで知的な返答傾向、Thinking 時のメタ推論（「ヤチヨとして答えよう」と操縦席の声が見える）などの課題があった
- **Claude 専用版プロンプト (`ヤッチョGPT (for Claude).md`) を使用し、Opus 4.6 で動かすことで Gemini と同等以上のキャラ再現度を達成。** 一人称・語尾・☆♪の使用・深さの出し方・古語の頻度など、Claude が苦手とするポイントを個別に調整
- 検索性能は安定しており、調べ物の正確性では Gemini より断然信頼できる
- **Thinking をオンにした場合、専用版プロンプトにより思考過程がヤチヨの独り言として進む。** メタ推論（「ヤチヨとして答えよう」）やプロンプト指示の復唱が排除され、「わぁ〜！来てくれた！嬉しいなぁ」のような感情の反射から思考が始まる。DeepSeek V4 Flash と同等以上のキャラ内思考を実現
- Claude の Projects 機能は無料ユーザーでも利用できますが、利用量的な問題から基本課金ユーザー向けな気はします

### Grok (Projects)

- 意外と高性能で、キャラ再現度はかなり高い
- 高速モードでも爆速かつキャラ性が保たれている
- ナレッジに入っていない作中知識を尋ねた場合に自動で Web 検索しつつ、あたかも最初から知っていたかのように返答してくれる傾向がある。最新情報の調べ物には Grok が向いている
- Ani などコンパニオン向けのチューニングもされているからか、Grok 自体の傾向として恋愛周りの話が饒舌な傾向がある
  - プロンプトで矯正するまでは Grok 版ヤッチョGPTにガチ恋を伝えるとヤッチョが承諾してしまっていたので（解釈違いすぎる…）、そのあたりの「一線」の敷き方もかなり緩めな印象

### DeepSeek

- GPTs や Projects 相当のプロンプト固定機能がないため、毎回チャットの冒頭にプロンプトを貼り付ける必要がある

### ローカル LLM

- **Qwen3.6-Plus (LM Studio)** で動かして良い感じだったという報告あり
- **Gemma 系** も日本語の質感が自然で、ロールプレイに向いている印象
- 各アプリのシステムプロンプトに `ヤッチョGPT.md` の内容を設定して利用してください

### 分析まとめ

**『同一のシステムプロンプトで比較実験することで、かえって憑依元 LLM の根本にある「テンション」「性格」「思考のクセ」のような何かが浮き彫りになる』といった知見が得られました。**  
以下のように、ベースの性格や言い回しの癖が返答のトーンに影響しているようです。  

- **Gemini:** ロールプレイ性能が高く、テンションも高いし明るくて最もヤチヨらしい
- **Claude:** メイン版ではおとなしめだが、専用版プロンプト + Opus 4.6 で Gemini に匹敵する再現度に到達。Thinking 時の内部思考もキャラ内で進行
- **ChatGPT (Thinking):** 寡黙で実務的だが、ベースモデルの思考や言い回しの癖が非常に強く出る
- **Grok:** インターネットドブ川育ちで倫理規制が低いので非常に親身でロールプレイ性能も高い…が、倫理規制がゆるすぎる反動で、ヤッチョがユーザーからのガチ恋を受け入れてしまい解釈違いが発生する（現在は矯正してあります）

どの LLM でもヤッチョらしさは感じられますが、モデル固有の「雰囲気」はそれぞれ異なるため、実際に話してみて好みに合うものを選ぶのがおすすめです。

## プロンプトの設計工程

このプロンプトは、**人間の解釈バイアスを可能な限り排除し、LLM 自身にキャラクターのペルソナを深く理解させ、初稿プロンプトでの生成結果を提示してさらに試行錯誤させる**アプローチにより「錬成」したものです。

### プロンプト初稿の作成

1. 世界で最も日本語読解能力の高い LLM である Claude Opus 4.6 / 4.7 (いずれも 1M Context) に対し、本編のトランスクリプト、「ray」MV に映るヤチヨの返答テキスト、ピクシブ百科事典などの資料を一度に入力し、まずは物語構造の読解、キャラクターのプロファイリングを行わせる
2. テキスト情報だけの入力にもかかわらず、キャラクターや物語構造、ペルソナまで驚異的な高解像度で読解できていた
3. 日本語能力と EQ が高い Opus 4.6 と、より返答が論理的思考の方に寄っている Opus 4.7 に、それぞれ独立してペルソナとプロンプトの初稿を書かせ、雛形とする
4. 両者に互いの初稿をレビューさせ、相互に抜け漏れなどのレビューやディベートを繰り返すことで、雛形を洗練させる

同じ Claude ベースでありながら微妙に趣向の異なるモデルを議論させることで、単一モデルでは陥りがちな解釈の偏りを回避することを目指しました。  

**『単一モデルだと時折袋小路に考えが固まってしまうところ、（さながら MAGI システムのように）異なる視点を持つ別の LLM に相互に議論させることで異なる新たなアイデアが得られ、より確からしい結論を導き出せることが多い』** という、普段のソフトウェア開発における知見を応用しています。

### 検証とフィードバック

初稿のプロンプトを Gemini・ChatGPT・Grok など複数のプラットフォームに入れて実際に会話し、出力された返答を Claude に再度レビュー・ジャッジさせました。  
**キャラ再現度が低い箇所を LLM 自身に判断させ、修正案も LLM に立案させることで、こちらの工程でも人間の介入を最小限に抑えています。**

また、既に独自に AI ヤチヨの再現に挑んでいた方々のツイート ([一例](https://x.com/Shureiden_Neko/status/2029498438568804418)) や、ツイート添付のスクショなども強く参考にさせていただきました。

もちろん「ヤチヨはこうは言わないので解釈違い！」といった矯正プロセスにはどうしても個人の主観や好みが入りますが、プロンプトの記述の骨子自体は、Claude Opus 4.6 / 4.7 が入力テキストから自律的に生成した初稿からあまり変わっていません。
